Map Elements

Intro

The Map Element is a standard component for all executable Elements in your Flow applications. As a result, the Map Element serves many purposes. It can be used to presend Pages to the user, to execute decisions and logical operations, call external APIs as messages to third party applications and manipulate data by saving, loading and deleting. The Map Element therefore forms the core logic of your Flow applications and it's how you connect the Map Elements together that make the application function.

Save

To save (create or update) a Map Element object, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/flow/{flowId}/{editingToken}/element/map POST Object: Map Element Request Object Object: Map Element Response Object

Where the parameters are as follows:

Property Type Description
flowId String: GUID The unique identifier for the Flow you wish to add the Map Element to.
editingToken String: GUID The unique identifier for the editing session of the Flow.

Query

To query for Map Element objects, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/flow/{flowId}/{editingToken}/element/map?filter={filter} GET N/A Array: Map Element Response Objects

Where the parameters are as follows:

Property Type Description
flowId String: GUID The unique identifier for the Flow holding the Map Elements.
editingToken String: GUID The unique identifier for the editing session of the Flow.
filter String The filter can currently supports two possible options - search by exact developer name or search by partial developer name. To search for an exact developer name use: filter=developerName eq 'name of your element' To search for by partial developer name use: filter=substringof(developerName,'partial name of your element') This follows the conventions established in the OData standard, though we do not currently support the $ prefix.

Delete

To delete a specific Map Element object, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/flow/{flowId}/{editingToken}/element/map/{id} DELETE N/A N/A

Where the parameters are as follows:

Property Type Description
flowId String: GUID The unique identifier for the Flow holding the Map Element.
editingToken String: GUID The unique identifier for the editing session of the Flow.
id String: GUID The unique identifier for the Map Element you wish to delete. This will not delete the element in previous versions - it will only delete the Map Element from builds of your Flow done from this point forward.

You must also provide the draw API with the following header value:

Name Value
Authorization The authorization String returned from logging into the api/draw/1/authentication API.
DELETE https://flow.manywho.com/api/draw/1/flow/104acb42-c01f-419f-b4bf-fccf867d166c/2d300d10-dd8c-43d7-abd6-889565a33c50/element/map/jf9s14c4-3344-f9s1-2kdd-fis94b3e9djw

JSON Objects Object structures

Map Element Request Object

The map element request object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the Map Element. The id should be null for "insert" requests and a valid identifier for "update" requests. This property is created by the service.
elementType String: ElementType The type of element this metadata represents.
developerName String The developer name for the Map Element. This is useful for keeping track of the Map Element in the modeling tool and API.
developerSummary String (Optional) The developer summary the author provided to give more information about the Map Element.
groupElementId String: GUID (Optional) The unique identifier for the Group Element that contains this Map Element. If a Map Element is inside a Group, it inherits certain behaviours of the parent Group. For example, a SWIMLANE Group wraps all child Map Elements in a security context.
x Integer The x location of the Map Element on the Flow diagram.
y Integer The y location of the Map Element on the Flow diagram.
outcomes Array: Outcome Objects The list of outcomes that are available for this Map Element. An Outcome is used to connect the flow of execution from one Map Element in the Flow to another. An Outcome can take the form of a Page button, but also define system steps such as rules.
pageElementId String: GUID The unique identifier for the Page Element that should be rendered when this Map Element executes. For simple steps, use the userContent property.
operations Array: Operation Objects The list of operations that should be performed when this Map Element executes. Operations are used to change the value of Value Elements in the executing Flow (State).
viewMessageAction Object: View Message Action Object The view message action object defines the interface inputs/outputs for calling against a REMOTE_PAGE Map Element. For the REMOTE_PAGE, the page view is delivered by an external system - allowing developers to hand-code very complex UI scenarios. This is not yet supported by the engine.
messageActions Array: Message Action Objects The list of message actions that should be executed when this Map Element executes. The message action objects define the interface of inputs/outputs for calling against each Service message.
dataActions Array: Data Action Objects The list of data actions that should be executed when this Map Element executes. The data action objects define the Values and bindings that should be used to perform CRUD operations against each Service and Value.
postUpdateToStream Boolean Indicates if this Map Element should post an update to the collaboration stream.
postUpdateMessage String The content of the message that should be posted to the collaboration stream.
postUpdateWhenType String: PostUpdateWhenType The point at which the post should be made to the collaboration stream.
userContent String The content that should be shown to the user at this step in the Flow. This property should only be used for very simple Flows and informational UI. For anything more than simple messaging, use the Page Element and associate it with this Map Element using the pageElementId property.
statusMessage String The content that should be shown to the user while waiting for a system step to complete.
notAuthorizedMessage String The content that should be shown to the user if they are not authorized to take action on this Map Element.
updateByName Boolean Indicates if the platform should attempt to find a Type Element with the same developer name as the one provided and match them up by name as opposed to 'id'. This is useful when creating scripts to create Flows - as you can use the developerName property as the reference as opposed to needing to know the ids of all created Elements.

Map Element Response Object

The Map Element has all of the same properties as the request object, but additionally the properties listed below:

Property Type Description
dateCreated String: Date/Time in JavaScript 1.8.5/ECMAScript 5th Edition format The date the Map Element was first created.
dateModified String: Date/Time in JavaScript 1.8.5/ECMAScript 5th Edition format The date the last modificiation was made to the Map Element.
userCreated String: GUID The unique identifier for the user who created the Map Element.
userModified String: GUID The unique identifier for the user who last modified the Map Element.
userOwner String: GUID The unique identifier for the user who owns this Map Element.

Outcome Object

The outcome object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the outcome. This property is created by the service.
developerName String The developer name to help identify this outcome in tooling and APIs.
developerSummary String (Optional) The developer summary the author provided to give more information about the Map Element.
label String (Optional) The label that should appear with the outcome. For UI situations, this is typically the text that will appear on the button.
nextMapElementId String: GUID (Optional) The unique identifier for the next Map Element in the Flow that should be executed if this outcome is selected.
pageActionType String: PageActionType (Optional) The page action type can be used to provide additional metadata to help player developers provide standard styling and positions for outcomes. The enumeration provided is partly implemented by the existing players, though the values can be extended as needed.
isBulkAction Boolean Indicates if this is bulk action. This is commonly used when presenting lists of object data to tell the player if the outcome should be displayed inline with each object entry or if it should be displayed at the top of the component as a "bulk" action.
pageActionBindingType String: PageActionBindingType (Optional) The page action binding type is bound to functionality in the flow and instructs the engine what to do with any data provided in the page components.
pageObjectBindingId String: GUID (Optional) The unique identifier for the page container or component this outcome should be bound to. If you bind an outcome to a container or component, it can also indicate additional functionality: e.g. isBulkAction, etc. It also helps player designers lay out the page.
comparison String: Comparison Object (Optional) The comparison object defines the rules that should be evaluated for this outcome to be selected by the system. As a result, the comparison object should only be used for system level outcomes.
order Integer The order in which the outcomes should be rendered relative to its peers. The lowest number is rendered first.

Comparison Object

The comparison object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the operation. This property is created by the service.
comparisonType String: ComparisonType The comparison to use when evaluating the immediate child rules associated with this comparison object.
rules Array: Rule Objects The list of rules that need to be evaluated for this outcome to be selected.
comparison Array: Comparison Objects The list of child comparisons that need to be evaluated for this outcome to be selected. The comparison object is recursive allowing rules to be nested: e.g. ({a} > {b} OR {c} <= {d}) AND {b} == {f}
order Integer The order in which the comparison should be evaluated with respect to its peers. The lowest number is evaluated first.

Rule Object

The rule object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the rule. This property is created by the service.
leftValueElementToReferenceId Object: Value Element Id Object The reference to the Value Element that should be used for the "left" side of the rule evaluation: e.g. if {left} is greater than {right} then ...
criteriaType String: CriteriaType The criteria that should be used when evaluating this rule: e.g. EQUAL
rightValueElementToReferenceId Object: Value Element Id Object The reference to the Value Element that should be used for the "right" side of the rule evaluation: e.g. if {left} is greater than {right} then ...

Operation Object

The operation object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the operation. This property is created by the service.
valueElementToApplyId Object: Value Element Id Object The Value Element that should be assigned as part of this operation.
valueElementToReferenceId Object: Value Element Id Object The Value Element that should be used to source the value that will be applied to this Value Element.
order Integer The order in which the operations should be executed. The lowest number is executed first.

Message Action Object

The message action object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the message action. This property is created by the service.
developerName String The developer name to help identify this message action in tooling and APIs.
serviceElementId String: GUID The unique identifier for the Service Element that this message action will be executed against. The Service Element also defines the list inputs/outputs that need to be configured for this message action.
uriPart String The unique Uri part that should be used when invoking the Service Element. The Uri part will inform the Service implementation which message action you are executing against.
inputs Array: Message Input Object The list of message inputs that should be sent to the Service when this action is invoked. The list of inputs is defined by the Service Element.
outputs Array: Message Output Object The list of message outputs that should be sent back from the Service and applied to Value Elements in the executing Flow (State). The list of outputs is defined by the Service Element.

Message Input Object

The message input object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the message input. This property is created by the service.
developerName String The developer name for the input defined by the Service Element definition of this message action. This name will be used by the Service Implementation to match which value is being assigned.
valueElementToReferenceId Object: Value Element Id Object The reference to the Value Element that should be used to provide the value for this input.

Message Output Object

The message output object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the message output. This property is created by the service.
developerName String The developer name for the output defined by the Service Element definition of this message action. This name will be used by the Service Implementation to match which value is being sent back.
valueElementToApplyId Object: Value Element Id Object The reference to the Value Element that should assigned by the mapped output value.

Data Action Object

The data action object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the data action. This property is created by the service.
developerName String The developer name to help identify this data action in tooling and APIs.
serviceElementId String: GUID The unique identifier for the Service Element that this data action will be executed against.
crudOperationType String: CrudOperationType The CRUD (create, read, update, delete - or in ManyWho - SAVE, LOAD, DELETE) operation that should be performed on the objects provided in this data action.
valueElementToReferenceId Object: Value Element Id Object The reference to the Value Element that should be used to send data to the Service. The Value Element must be a ContentObject or ContentList and must have a valid binding in the selected Service Element.
valueElementToApplyId Object: Value Element Id Object The reference to the Value Element that should be used to apply any data back from the Service. The Value Element must be a ContentObject or ContentList and must have a valid binding in the selected Service Element.
objectDataRequestProperties Object: Object Data Request Properties Object The object data request properties object holds the specific configuration information that the Service can use to optimize requests from ManyWho.

Object Data Request Object

The object data request object properties are as follows:

Property Type Description
typeElementId String: GUID The unique identifier for the Type Element that will define the structure of the data returned by this request.
typeElementBindingId String: GUID The unique identifier for the binding that should be used on the provided Type Element.
typeElementPropertyId String: GUID The unique identifier for the property in that should be displayed in this column. This pertains to the Type for the bound data or the object data request - not the underlying value binding.
listFilter Object List Filter Object The filter that should be applied to the data at runtime. The list filter can be altered by events on the page. The list filter object can also be used by the player to support additional component filtering and logic such as paging. The properties of the filter can therefore be altered at runtime to optimize the user experience.
command Object: Command This property is not yet supported but will allow additional runtime options such as inline search.

List Filter Object

The list filter object properties are as follows:

Property Type Description
filterId Object: Value Element Id Object The reference to the Value Element that holds a very specific identifier to filter the objects by. This property should be used when loading specific objects as opposed to lists of object results.
comparisonType String: ComparisonType The comparison to use when evaluating the where objects associated with this filter.
where Array: List Filter Where Object The filter criteria that should be applied during the lookup.
orderByTypeElementPropertyId String: GUID The unique identifier for the property in the Type Element (associated with this filter) that should be used for ordering the results.
orderByDirection String: orderByDirectionType The direction in which to order the results.
limit Integer The maximum number of results to return from the request.
filterByProvidedObjects Boolean If objects are provided from the Value Element specified in the valueElementToReferenceId these values can form the 'filter' that the remote Service should use. This can be used in situations where the objects need to be 'hydrated'.

List Filter Where Object

The list filter where object properties are as follows:

Property Type Description
columnTypeElementPropertyId String: GUID The unique identifier for the column in the bound component data to filter by.
criteriaType String: CriteriaType The criteria to filter the list data by.
valueElementToReferenceId Object: Value Element Id Object The reference to the Value Element that contains the value to filter the list data by.

Value Element Id Object

The value element id object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the Value to be referenced with this identifier.
typeElementPropertyId String: GUID The unique identifier for a Property in the Value to be referenced with this identifier. This only applies for Values of type ContentObject.
command String The command to execute as part of the Value reference. For certain objects, the command exposes additional properties that can be referenced.

API Enumerations Helpful values

CriteriaType Enumeration

The CriteriaType enum contains the following values:

Value Description
EQUAL {left} is equal to {right}
NOT_EQUAL {left} is not equal to {right}
GREATER_THAN {left} is greater than {right}. For ContentString types, this is done alphabetically.
GREATER_THAN_OR_EQUAL {left} is greater than or equal to {right}. For ContentString types, this is done alphabetically.
LESS_THAN {left} is less than {right}. For ContentString types, this is done alphabetically.
LESS_THAN_OR_EQUAL {left} is less than or equal to {right}. For ContentString types, this is done alphabetically.
CONTAINS {left} contains the characters in {right}. This is only valid for ContentString types and is case insensitive.
STARTS_WITH {left} starts with the characters in {right}. This is only valid for ContentString types and is case insensitive.
ENDS_WITH {left} ends with the characters in {right}. This is only valid for ContentString types and is case insensitive.
IS_EMPTY {left} is empty or null. When this criteria type is used, it must be compared with a {right} that is a ContentBoolean.

PageActionBindingType Enumeration

The PageActionBindingType enum contains the following values:

Value Description
SAVE The engine should apply all values collected to the state.
NO_SAVE The engine should ignore all values collected and not apply anything to the state except the outcome action.

ContentType Enumeration

The ContentType enum contains the following values:

Value Typed Description
ContentString No A Java java.lang.String or C# System.String value.
ContentNumber No A Java java.lang.Double or C# System.Double value.
ContentBoolean No A Java java.lang.Boolean or C# System.Boolean value.
ContentPassword No A Java java.lang.String or C# System.String value. If you indicate a value is a password, we use additional value encryption on storage.
ContentDateTime No A Java java.lang.DataTime or C# System.DateTime. Using the REST API, the value should be provided in JavaScript 1.8.5/ECMAScript 5th Edition format.
ContentContent No A Java java.lang.String or C# System.String value. If you indicate a value is "Content", we typically expect that the content holds HTML formatted String content. The engine will also parse the contents of this String with any value references.
ContentObject Yes The value is Object Data based on the ManyWho format. This allows you to pass complex data into ManyWho for processing, including child objects and lists. Any Object Data being passed into the system also needs to have a Type as defined by your Type Elements.
ContentList Yes The value is a list of Object Data based on the ManyWho format. This allows you to pass lists of complex data into ManyWho for processing (of the same Type), including child objects and lists. Any Object Data being passed into the system also needs to have a Type as defined by your Type Elements.
