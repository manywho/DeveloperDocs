Intro

Save

To save (create or update) a Group Element object, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/flow/{flowId}/{editingToken}/element/group POST Object: Group Element Request Object Object: Group Element Response Object

Where the parameters are as follows:

Property Type Description
flowId String: GUID The unique identifier for the Flow you wish to add the Group Element to.
editingToken String: GUID The unique identifier for the editing session of the Flow.

Query

To query for Group Element objects, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/flow/{flowId}/{editingToken}/element/group?filter={filter} GET N/A Array: Group Element Response Objects

Where the parameters are as follows:

Property Type Description
flowId String: GUID The unique identifier for the Flow holding the Group Elements.
editingToken String: GUID The unique identifier for the editing session of the Flow.
filter String The filter can currently supports two possible options - search by exact developer name or search by partial developer name. To search for an exact developer name use: filter=developerName eq 'name of your element' To search for by partial developer name use: filter=substringof(developerName,'partial name of your element') This follows the conventions established in the OData standard, though we do not currently support the $ prefix.

Delete

To delete a specific Group Element object, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/flow/{flowId}/{editingToken}/element/group/{id} DELETE N/A N/A

Where the parameters are as follows:

Property Type Description
flowId String: GUID The unique identifier for the Flow holding the Group Element.
editingToken String: GUID The unique identifier for the editing session of the Flow.
id String: GUID The unique identifier for the Group Element you wish to delete. This will not delete the element in previous versions - it will only delete the Group Element from builds of your Flow done from this point forward.

You must also provide the draw API with the following header value:

Name Value
Authorization The authorization String returned from logging into the api/draw/1/authentication API.
DELETE https://flow.manywho.com/api/draw/1/flow/104acb42-c01f-419f-b4bf-fccf867d166c/2d300d10-dd8c-43d7-abd6-889565a33c50/element/group/9c199d19-dd8c-54d7-cdd6-770565a44c20

JSON Objects Object structures

Group Element Request Object

The group element request object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the Group Element. The id should be null for "insert" requests and a valid identifier for "update" requests. This property is created by the service.
elementType String: ElementType The type of element this metadata represents.
developerName String The developer name for the Group Element. This is useful for keeping track of the Group Element in the modeling tool and API.
developerSummary String (Optional) The developer summary the author provided to give more information about the Group Element.
x Integer The x location of the Group Element on the Flow diagram.
y Integer The y location of the Group Element on the Flow diagram.
height Integer The height of the Group Element on the Flow diagram.
width Integer The width of the Group Element on the Flow diagram.
authorization Object: Group Authorization The Authorization object for this group.
updateByName Boolean Indicates if the platform should attempt to find a Group Element with the same developer name as the one provided and match them up by name as opposed to 'id'. This is useful when creating scripts to create Flows - as you can use the developerName property as the reference as opposed to needing to know the ids of all created Elements.

Group Element Response Object

The Group Element has all of the same properties as the request object, but additionally the properties listed below:

Property Type Description
dateCreated String: Date/Time in JavaScript 1.8.5/ECMAScript 5th Edition format The date the Group Element was first created.
dateModified String: Date/Time in JavaScript 1.8.5/ECMAScript 5th Edition format The date the last modificiation was made to the Group Element.
userCreated String: GUID The unique identifier for the user who created the Group Element.
userModified String: GUID The unique identifier for the user who last modified the Group Element.
userOwner String: GUID The unique identifier for the user who owns this Group Element.

Group Authorization Object

The group authorization object properties are as follows:

Property Type Description
serviceElementId String: GUID The unique identifier for the Service Element that this authorization configuration is associated. The Service Element must support identity.
globalAuthenticationType String: GlobalAuthenticationType The global authentication type for this Flow.
streamBehaviourType String: StreamBehaviourType The stream behaviour type for this Flow.
groups Array: Group Authorization Group Objects The list of groups that are associated with this authorization configuration.
groups Array: Group Authorization User Objects The list of users that are associated with this authorization configuration.
locations Array: Group Authorization Location Objects The list of locations that are associated with this authorization configuration. This is not yet supported by the service.

Group Authorization Group Object

The group authorization group object properties are as follows:

Property Type Description
authenticationId String The unique identifier to match the correct group. This is an external unique identifier based on the directory implementation of the Service.
attribute String: ENUM The unique attribute to match the correct group attribute. This enumeration is based on the directory implementation of the Service.

Group Authorization User Object

The group authorization user object properties are as follows:

Property Type Description
authenticationId String The unique identifier to match the correct user. This is an external unique identifier based on the directory implementation of the Service.
attribute String: ENUM The unique attribute to match the correct user attribute. This enumeration is based on the directory implementation of the Service.

API Enumerations Helpful values

GlobalAuthenticationType Enumeration

The GlobalAuthenticationType enum contains the following values:

Value Description
PUBLIC Indicates that the Group is public and therefore does not need to have a Service assigned to manage the authorization of users. However, if one is provided, it will be used and any identifying user information will be passed forward (e.g. location).
ALL_USERS Indicates that the Group is accessible to all users of a particular directory provided through the Service.
SPECIFIED Indicates that the Group has specific authorization rights that are defined in the groups, users or locations.

StreamBehaviourType Enumeration

The StreamBehaviourType enum contains the following values:

Value Description
USE_EXISTING Use an existing collaboration stream when the user enters this Group, or create a new one if one does not yet exist.
CREATE_NEW Create a new collaboration stream every time the user enters this Group.
NONE Do not provide a collaboration stream for this Group.
