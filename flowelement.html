Query

To query for Flow objects, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/flow?filter={filter} GET N/A Array: Flow Response Objects

Where the parameters are as follows:

Property Type Description
filter String The filter can currently supports two possible options - search by exact developer name or search by partial developer name. To search for an exact developer name use: filter=developerName eq 'name of your element' To search for by partial developer name use: filter=substringof(developerName,'partial name of your element') This follows the conventions established in the OData standard, though we do not currently support the $ prefix.

You must also provide the draw API with the following header value:

Delete

To delete a specific Flow object, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/flow/{id} DELETE N/A N/A

Where the parameters are as follows:

Property Type Description
id String: GUID The unique identifier for the Flow you wish to delete. This will also delete the previous versions from access - we can often recover your flow if you make a mistake deleting your flow.

You must also provide the draw API with the following header value:

Name Value
Authorization The authorization String returned from logging into the api/draw/1/authentication API.
DELETE https://flow.manywho.com/api/draw/1/flow/104acb42-c01f-419f-b4bf-fccf867d166c

JSON Objects Object structures

Flow Request Object

The flow request object properties are as follows:

Property Type Description
id Object: Flow Identifier Object The complete unique identifier for the currently edited version of the flow.
editingToken String: GUID The editing token for this particular editing session. An editing token is required so the modeling engine can track multiple user changes.
developerName String The developer name for the flow. When referencing flows by name, this is the name you should use in your referencing.
developerSummary String The developer summary the author provided to give more information about the flow.
startMapElementId String: GUID The unique identifier for the first element in the flow.
authorization Object: Group Authorization The Authorization object for this flow.

Flow Response Object

The flow response object has all of the same properties as the request object, but additionally the properties listed below:

Property Type Description
dateCreated String: Date/Time in JavaScript 1.8.5/ECMAScript 5th Edition format The date the Flow was first created.
dateModified String: Date/Time in JavaScript 1.8.5/ECMAScript 5th Edition format The date the last modificiation was made to the Flow.
userCreated String: GUID The unique identifier for the user who created the Flow.
userModified String: GUID The unique identifier for the user who last modified the Flow.
userOwner String: GUID The unique identifier for the user who owns this Flow.
isActive Boolean Indicates if this Flow version is the active version. In the case of run operations, this will always be true.
isDefault Boolean Indicates if this Flow version is the default version. In the case of run operations, this will always be true.

Group Authorization Object

The group authorization object properties are as follows:

Property Type Description
serviceElementId String: GUID The unique identifier for the Service Element that this authorization configuration is associated. The Service Element must support identity.
globalAuthenticationType String: GlobalAuthenticationType The global authentication type for this Flow.
streamBehaviourType String: StreamBehaviourType The stream behaviour type for this Flow.
groups Array: Group Authorization Group Objects The list of groups that are associated with this authorization configuration.
groups Array: Group Authorization User Objects The list of users that are associated with this authorization configuration.
locations Array: Group Authorization Location Objects The list of locations that are associated with this authorization configuration. This is not yet supported by the service.

Group Authorization Group Object

The group authorization group object properties are as follows:

Property Type Description
authenticationId String The unique identifier to match the correct group. This is an external unique identifier based on the directory implementation of the Service.
attribute String: ENUM The unique attribute to match the correct group attribute. This enumeration is based on the directory implementation of the Service.

Group Authorization User Object

The group authorization user object properties are as follows:

Property Type Description
authenticationId String The unique identifier to match the correct user. This is an external unique identifier based on the directory implementation of the Service.
attribute String: ENUM The unique attribute to match the correct user attribute. This enumeration is based on the directory implementation of the Service.

Flow Identifier Object

The flow identifier object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the flow. This identifier does not change for the lifetime of the flow and is generated when the flow is first created.
versionId String: GUID The unique identifier for the specific version of a flow. This identifier changes any time a change is made to the flow - in structure or content.

API Enumerations Helpful values

GlobalAuthenticationType Enumeration

The GlobalAuthenticationType enum contains the following values:

Value Description
PUBLIC Indicates that the Flow is public and therefore does not need to have a Service assigned to manage the authorization of users. However, if one is provided, it will be used and any identifying user information will be passed forward (e.g. location).
ALL_USERS Indicates that the Flow is accessible to all users of a particular directory provided through the Service.
SPECIFIED Indicates that the Flow has specific authorization rights that are defined in the groups, users or locations.

StreamBehaviourType Enumeration

The StreamBehaviourType enum contains the following values:

Value Description
USE_EXISTING Use an existing collaboration stream when the user enters this flow, or create a new one if one does not yet exist.
CREATE_NEW Create a new collaboration stream every time the user enters this flow.
NONE Do not provide a collaboration stream for this Flow.
