Add To Flow

In order to use a Service Element in your Flow, you must 'add' or 'import' the element to the Flow so our engine can keep track of it as a dependency. To add a reference to a specific Service Element object to your Flow, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/element/flow/{flowId}/service/{id} POST N/A N/A

Where the parameters are as follows:

Property Type Description
flowId String: GUID The unique identifier for the Flow you wish to add the Service Element to.
id String: GUID The unique identifier for the Service Element to be added to the Flow.

You must also provide the draw API with the following header value:

Name Value
Authorization The authorization String returned from logging into the api/draw/1/authentication API.
POST https://flow.manywho.com/api/draw/1/element/flow/7ba7a6a1-d6ba-4806-8cc0-77be8504e86d/service/31f7b173-4089-41e0-9d78-e55ed8a079c0

Query

To query for Service Element objects, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/element/service?filter={filter} GET N/A Array: Service Element Response Objects

Where the parameters are as follows:

Property Type Description
filter String The filter can currently supports two possible options - search by exact developer name or search by partial developer name. To search for an exact developer name use: filter=developerName eq 'name of your element' To search for by partial developer name use: filter=substringof(developerName,'partial name of your element') This follows the conventions established in the OData standard, though we do not currently support the $ prefix.

You must also provide the draw API with the following header value:

Delete

To delete a specific Service Element object, you can use the following API:

URL HTTP Verb JSON Request JSON Response
api/draw/1/element/service/{id} DELETE N/A N/A

Where the parameters are as follows:

Property Type Description
id String: GUID The unique identifier for the Service Element you wish to delete. This will not delete the element in previous versions - it will only delete the service element from builds of your Flow done from this point forward.

You must also provide the draw API with the following header value:

Name Value
Authorization The authorization String returned from logging into the api/draw/1/authentication API.
DELETE https://flow.manywho.com/api/draw/1/element/service/31f7b173-4089-41e0-9d78-e55ed8a079c0

JSON Objects Object structures

Service Element Request Object

The service element request object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the service element. The id should be null for "insert" requests and a valid identifier for "update" requests. This property is created by the service.
elementType String: ElementType The type of element this metadata represents. For service elements, the only valid value currently is: SERVICE
developerName String The developer name for the service element. This is useful for keeping track of the service element in the modeling tool and API.
developerSummary String (Optional) The developer summary the author provided to give more information about the service element.
uri String: Uri The location of the service implementation for the platform to callout against.
format String The REST messaging format to use to communicate with this service. Currently the only valid value for this property is: json
configurationValues Array: Service Value Request Objects The list of configuration value mappings the service needs to function. Each entry provides a reference to a Value in the Flow that contains the configuration value needed by the service plugin.
providesLogic Boolean Indicates if the service provides functionality for 'logic'. Logic allows authors to make API calls as part of elements that support messaging: 'Message', 'Page', and 'Remote Page' currently. Messaging is used for asynchronous and synchronous use-cases.
providesViews Boolean Indicates if the service provides functionality for 'views'. A View allows authors to build Flows that include 'Remote Page' elements - e.g. the UI of the page is not actually hosted on the ManyWho platform, but rather the UI is provided by the external service.
providesContent Boolean Indicates if the service provides functionality for 'content'. Content allows the author to reference files and content from the service as dynamic references - meaning that the files and content can be managed outside of ManyWho, but embedded in your Flows.
providesDatabase Boolean Indicates if the service provides functionality for 'database'. Database functionality allows the author to map their Typed Objects and Lists back to this service for storage. The service then acts as the persistence implementation to store and retrieve the data as neede by the Flow.
providesIdentity Boolean Indicates if the service provides functionality for 'identity'. Identity functionality allows the author to manage permissions to their Flows and sections of their Flows (via Swimlanes) using this service as the directory. This also allows users to login to the Flows using the information stored in this service directory.
providesSocial Boolean Indicates if the service provides functionality for 'social'. Social allows the author to add features for feed, file and user collaboration to their Flows using this service as the underlying social network.
providesLocation Boolean Indicates if the service provides functionality for 'location'. ManyWho optionally collects location information about the user when running a Flow. This information can be used to make decisions about permissions and also data filtering - though it is up to the service to implement this functionality.
actions Array: Service Action Request Objects The list of 'logic' operations that are available for the service. If the underlying service 'providesLogic', this is where the 'interface' for those logic operations should be stored. This allows the author to know what inputs and outputs are provided by the actions provided by this service.
install Object: Service Install Request Objects As part of the service creation, you can provide additional installation metadata that will be created as part of the service. This object defines the metadata that this service wants to create as part of the creation of the service.
updateByName Boolean Indicates if the the platform should attempt to find a service with the same developer name as the one provided and match them up by name as opposed to 'id'. This is useful when creating scripts to create flows - as you can use the developerName property as the reference as opposed to needing to know the ids of all created elements.

Service Value Request Object

The service value request object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the service value. This property is created by the service.
developerName String The developer name for the service value. This needs to match up with the developerName of the configuration value sent back from the service description.
valueElementToReferenceId Object: Value Element Id The compound identifier and command for the reference Value to be used to provide the actual value for the configuration value. The Value must be the same ContentType and TypeElement definition (if an Object or List) as that provided by the service description.
contentValue String The hard-coded value for the configuration value. This property is not yet supported and may be part of future versions of the ManyWho platform.
contentType String: ContentType They type of content the component holds.

Service Action Request Object

The service action request object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the service action. This property is created by the service.
uriPart String The uri part that will be used in addition to the base uri provided for the service. This needs to match up with the uriPart of the actions sent back from the service description.
developerName String The developer name for the service action. This does not need to match with the service description as the matching to action is done based on uri part above.
developerName String The developer name for the service action. This does not need to match with the service description as the matching to action is done based on uri part above.
developerSummary String (Optional) The developer summary the author provided to give more information about the service action.
isViewMessageAction Boolean This property is not yet supported. Indicates if this particular service action will return a View as opposed to being a system-to-system API reference. If this is a View message action, then it can be used in 'Remote Page' map elements.
serviceActionOutcomes Array: Service Action Outcome This property is not yet supported. Provides the outcomes that the View exposes for the Author to use as outcomes in their Flow.
serviceInputs Array: Service Value Request Objects The list of input values that are available for this particular action. The only supported properties on the service value request object are: id, developerName and contentType as the Value references are defined by the actions in each 'Message' map element.
serviceOutputs Array: Service Value Request Objects The list of output values that are available for this particular action. The only supported properties on the service value request object are: id, developerName and contentType as the Value references are defined by the actions in each 'Message' map element.

Service Install Request Object

The service install request object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the service action. This property is created by the service.
typeElements Array: Type Element Request Object The list of types can will be installed as part of this service creation. Once the service element saves, the install object is no longer used as it will create all the Type elements as part of the single save operation.

Value Element Id Object

The value element id object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the Value to be referenced with this identifier.
typeElementPropertyId String: GUID The unique identifier for a Property in the Value to be referenced with this identifier. This only applies for Values of type ContentObject.
command String The command to execute as part of the Value reference. For certain objects, the command exposes additional properties that can be referenced.
Property Type Description
dateCreated String: Date/Time in JavaScript 1.8.5/ECMAScript 5th Edition format The date the flow was first created.
dateModified String: Date/Time in JavaScript 1.8.5/ECMAScript 5th Edition format The date the last modificiation was made to the flow.
userCreated String: GUID The unique identifier for the user who created the flow.
userModified String: GUID The unique identifier for the user who last modified the flow.
userOwner String: GUID The unique identifier for the user who owns this flow.
editingToken String: GUID This property is returned, but is always 'null' for run operations.
id Object: Flow Identifier Object The complete unique identifier needed to initialize and run the flow.
developerName String The developer name for the flow. If you are referencing flows by name, this is the name you should use in your referencing.
developerSummary String The developer summary the author provided to give more information about the flow.
isActive Boolean Indicates if this flow version is the active version. In the case of run operations, this will always be true.
isDefault Boolean Indicates if this flow version is the default version. In the case of run operations, this will always be true.
startMapElementId String: GUID The unique identifier for the first element in the flow.
authorization Object: Authorization The Authorization object for this flow.

Engine Initialization Request Object

The engine initialization request object properties are as follows:

Property Type Description
flowId Object: Flow Identifier Object The complete unique identifier for the flow you want to initialize.
annotations Map (Optional) Key value pairs you wish to annotate to the flow. Annotations take the form of {"mykey":"myvalue"}. Any annotations added to the state will be persisted for the duration of the flow. Annotations are passed to the executing player and also through to plugin services. Annotations can be changed at any time through the execution of the flow.
inputs Array: Engine Value Objects (Optional) An array of engine value objects that will be used to assign values in the flow at initialization.
mode String: UrlPlayerUrl (Optional) The location of the player that should be used for sharing and notifications when first running the flow. The service will automatically append the "flow-id" parameter to this url so the player knows which flow it is playing.
mode String: UrlJoinPlayerUrl (Optional) The location of the player that should be used for sharing and notifications when joining a running flow. The service will automatically append the "join" parameter to this url so the player knows which flow and state it is playing (the "join" parameter is the state identifier).
mode String: Mode (Optional) The mode you wish to run the flow in. The mode is mainly useful for debugging purposes as you can step through the flow and also view state information to check everything is working as expected.

Engine Initialization Response Object

The engine initialization response object properties are as follows:

Property Type Description
stateId String: GUID The unique identifier for the initialized flow state. The state identifier is needed throughout the execution of the flow as it is the pointer that points the engine to the correct running instance of a flow.
stateToken String: GUID The unique identifier for the token response from the state. The state token identifier changes with every cycle of request/response. The token is needed as it tells the engine how in-sync your request is with the current service side state.
currentMapElementId String: GUID The unique identifier for the map element the user is currently executing against. When initializing the flow, this will be the start element for the flow if the user has authenticated correctly, otherwise, this will be null.
currentStreamId String: GUID The unique identifier for the collaboration stream associated with this flow. This identifier allows you to make calls against the "social" APIs to get the posts and comments associated with the flow. The stream identifier can change as you progress through the flow.
statusCode String The values mimic standard REST codes, but as a String. A "200" indicates the user is authenticated to execute the flow. A "401" indicates the user needs to login based on the authorization context information provided in the response.
authorizationContext Object: Authorization Context Object The authorization context for the initialization. This object will tell you how to login to the correct system to begin executing the flow.

Engine Invoke Request Object

The engine invoke request object properties are as follows:

Property Type Description
stateId String: GUID The unique identifier for the initialized flow state. The state identifier is needed throughout the execution of the flow as it is the pointer that points the engine to the correct running instance of a flow.
stateToken String: GUID The unique identifier for the token response from the state. The state token identifier changes with every cycle of request/response. The token is needed as it tells the engine how in-sync your request is with the current service side state.
currentMapElementId String: GUID The unique identifier for the map element the user is currently executing against.
invokeType String: InvokeType The way you want to invoke the engine as part of this request. The user may be navigating forward, or you may be wanting to perform a sync operation to update the UI due to another user making a change.
annotations Map (Optional) Key value pairs you wish to annotate to the flow. Annotations take the form of {"mykey":"myvalue"}. Any annotations added to the state will be persisted for the duration of the flow. Annotations are passed to the executing player and also through to plugin services. Annotations can be changed at any time through the execution of the flow.
geoLocation Object: Geo Location Object (Optional) An object representing the current location information of the calling user.
mapElementInvokeRequest Object: Map Element Invoke Request Object The map element invoke request holds the data that was collected from the user. This can be inputs from a form or simply the selected outcome (e.g. the button they clicked to move forward).
mode String: Mode (Optional) The mode you wish to run the flow in. The mode is mainly useful for debugging purposes as you can step through the flow and also view state information to check everything is working as expected.

Map Element Invoke Request Object

The map element invoke request object properties are as follows (these are the properties we're exposing currently and the documentation will be expanded shortly):

Property Type Description
selectedOutcomeId String: GUID The unique identifier for the outcome the user is taking action on. The outcomes are provided in the Map Element Invoke Response Object as part of the outcomeResponses property.

Engine Invoke Response Object

The engine invoke response object properties are as follows:

Property Type Description
stateId String: GUID The unique identifier for the initialized flow state. The state identifier is needed throughout the execution of the flow as it is the pointer that points the engine to the correct running instance of a flow.
stateToken String: GUID The unique identifier for the token response from the state. The state token identifier changes with every cycle of request/response. The token is needed as it tells the engine how in-sync your request is with the current service side state.
alertEmail String The email address that should be used to notify of any Flow or player related faults. This email address is typically the email of the author of the Flow.
waitMessage String The message that should be shown to the user if the engine returns an Invoke Type of 'WAIT'.
notAuthorizedMessage String The message that should be shown to the user if the engine returns an Invoke Type of 'STATUS'. The engine will automatically handle UI permissions - so this user will not recieve Page metadata.
currentMapElementId String: GUID The unique identifier for the map element the user is currently executing against.
currentStreamId String: GUID The unique identifier for the collaboration stream the user is currently using to collaborate with colleagues. If the Flow is not leveraging collaboration streams, this property will be null or blank.
invokeType String: InvokeType The way the engine responded to your initial invoke request.
annotations Map (Optional) Key value pairs you wish to annotate to the flow. Annotations take the form of {"mykey":"myvalue"}. Any annotations added to the state will be persisted for the duration of the flow. Annotations are passed to the executing player and also through to plugin services. Annotations can be changed at any time through the execution of the flow.
mapElementInvokeResponses Array: Map Element Invoke Response Objects The list of map element invoke responses generated by the executed element or elements in the flow. Currently the array contains only the one object, which is the object that is the current element being executed in the flow.
stateLog N/A This property is not currently supported. In future versions, this will contain the execution history of the flow.
preCommitStateValues Array: Engine Value Objects The list of engine values that are currently in a pre-commit state. This is only populated when the engine is being run in "DEBUG" or "DEBUG_STEPTHROUGH" mode. Values are only committed to the state after the element has completed execution successfully (usually as a result of the user invoking the engine "FORWARD" and the page validating correctly).
stateValues Array: Engine Value Objects The list of engine values that are currently committed in the state. This is only populated when the engine is being run in "DEBUG" or "DEBUG_STEPTHROUGH" mode.
outputs Array: Engine Value Objects The list of engine values for all values in the flow that are set as OUTPUT or INPUT/OUTPUT. This property is only populated when the engine responds with an invokeType of "DONE".
statusCode String The values mimic standard REST codes, but as a String. A "200" indicates the user is authenticated to execute the flow. A "401" indicates the user needs to login based on the authorization context information provided in the response.
runFlowUri String: Url The Url that should be used to run a new instance of this flow with this player. For cross-platform situations, the run Url should redirect to the appropriate player for the calling device.
joinFlowUri String: Url The Url that should be used to join this instance of this this flow with this player. For cross-platform situations, the join Url should redirect to the appropriate player for the calling device.
authorizationContext Object: Authorization Context Object The authorization context for the initialization. This object will tell you how to login to the correct system to begin executing the flow.

Map Element Invoke Response Object

The map element invoke response object properties are as follows:

Property Type Description
mapElementId String: GUID The unique identifier for the map element this response pertains to.
pageResponse Object: Page Response Object The meta data object for the page. This object should be used to generate the UI that will be presented to the user. If the map element invoke response contains a pageResponse object, then the engine is expecting the user to take action to progress the state of the flow.

Page Response Object

The page response object properties are as follows:

Property Type Description
label String (Optional) The label for the page. This is often used as a title for the page.
pageContainerResponses Array: Page Container Response Object (Optional) The scaffolding for the page UI containers. The page containers are nested objects that give the complete heirarchy of containers that make up the layour of the UI. The engine will only return this array of objects for a "JOIN" or "FORWARD" invokeType. For all other types, we assume the player has the UI layout and therefore is only needing metadata and content/data updates. The engine will always provide the complete page scaffolding and then rely on the data responses to hide, display, disable, etc.
pageComponentResponses Array: Page Component Response Object (Optional) The UI component base structures. The data contained in this object will not change based on events. As with the page container response objects, this data is only provided when the engine is called with an invokeType of "JOIN" or "FORWARD".
pageContainerDataResponses Array: Page Container Data Response Object (Optional) The meta data for the page containers. The metadata tells the player if the container is visible, enabled, etc. This information is subject to change based on events performed by the UI and this array of objects will always be returned by the engine regardless of invokeType (assuming the user is authenticated and the element has UI).
pageComponentDataResponses Array: Page Component Data Response Object (Optional) The meta data and data for the page components. The metadata tells the player if the component is visible, enabled, etc, and also tells the player the content of the component. This information is subject to change based on events performed by the UI and this array of objects will always be returned by the engine regardless of invokeType (assuming the user is authenticated and the element has UI).
order Integer (Optional) The order in which this page element should appear. The order will always be zero for the page element as it is the root container.
outcomeResponses Array: Outcome Response Object (Optional) The outcome response objects are the actions that can be performed by the user to navigate the flow from element to element. Outcome responses can be bound to page components to add additional context to the action (e.g. the button should appear beside a field input, above a table of records, etc)
rootFaults Map (Optional) Key value pairs taking the form of {"errorCode":"errorMessage"}. The root faults are populated if a fault has occurred that is not attributed to a specific value on the page. A root fault is often due to an element in the flow experiencing a fault and the author of the flow has determined that the user should be navigated to this page to rectify the problem. If an error is attributed to a value in the state, it will be bound to the page component as a validation error message.

Page Container Response Object

The page container response object properties are as follows:

Property Type Description
id String: GUID The unique identifier for the page component. All page components have a property of pageContainerId that will match with this identifier to indicate that the component should be a child of this container.
containerType String: ContainerType The container type tells the player the kind of container that needs to be rendered. This is usually to indicate if the container has particular layour properties such as vertical, horizontal or inline flow - or that the container is a group of tabs. The container type is not restricted to the enumeration provided here - however - all of the standard players provided by ManyWho are able to render the container types in the enum. If you wish to add more container types, be aware that you will need to update the player to support these.
developerName String The name the author has given for this container.
label (Optional) String The label for the container. This is often used as a title for the section of the page represented by this container.
pageContainerResponses Array: Page Container Response Object (Optional) The child scaffolding for this page container.
order Integer The order in which this container should appear on the page with respect to other page containers at the same level in the page container hierarchy.

Page Component Response Object

The page component response object properties are as follows:

Property Type Description
pageContainerDeveloperName String The developer name for the page container this component should be placed into. When rendering a UI, it's best to reference the pageContainerId as this developer name is not guaranteed to be unique.
pageContainerId String: GUID The unique identifier for the page container this component should be placed into. When rendering a UI, this is the best reference to use as the developer names are not guaranteed to be unique.
id String: GUID The unique identifer for this page component.
developerName String The name the author has given for this component.
componentType String: ComponentType The type of component that should be used. The platform currently supports the standard components such as "INPUT", "SELECT", etc as would be familiar with HTML UI. If you want to provide additional UI componentry, we recommend using "tags" to inform the player that you want to adapt the component based on the data in the tag. For example, you may want to change a particular number input to a "slider" or you might bind this page component to a widget - again, based on the data in the tag.
contentType String: ContentType They type of content the component holds.
label (Optional) String The label for the component. This is often the label for a form field.
columns (Optional) Array: Page Component Column Objects The columns associated with this page component. This property is only used for "TABLE"/"SELECT" components and details out the columns that should be displayed to the user. This property is not currently fully documented until we release the page composition tool.
size (Optional) Integer The size of the component. For "INPUT" components, this is typically the number of characters wide to display the input box.
maxSize (Optional) Integer The maximum size of characters the component can hold. For "INPUT" and "TEXTAREA" components, this is the amount of content these inputs can hold.
height (Optional) Integer The height of the component. For "TEXTAREA" components, this is typically the number of characters high to display the text box. For "IMAGE" components, this is the height in pixels of the image.
width (Optional) Integer The width of the component. For "TEXTAREA" components, this is typically the number of characters wide to display the text box. For "IMAGE" components, this is the width in pixels of the image.
hintValue (Optional) String The information that should be shown to the user to help them understand what they need to enter. For "INPUT" components, this if often the text you see in the input box (e.g. Enter Name) that disappears once you enter something into the input.
helpInfo (Optional) String The help information to assist the user in completing this page component. The help information is often something the user sees if they click on a "help" button beside the component or hover over a "help" icon.
order Integer The order in which this component should be placed with respect to other components in the same page container.
isMultiSelect (Optional) Boolean This property indicates if the component supports selection of more than one value. This is only applicable for "TABLE"/"SELECT" components.
hasEvents Boolean Indicates if changes to this component should result in a "SYNC" invoke type with the service. If the component has events, it's likely that the page state needs to change based on business rules. If the component does not have events, there is no need to update the service of changes made to the value of the component.

Engine Value Object

The engine value object properties are as follows:

Property Type Description
id String: GUID (Optional) The unique identifier for the value in the flow being assigned. Using the id to reference the value ensures that your player is guaranteed to be assigning the correct value. If you reference a value by developerName, it is possible for the author to break integration points simply by changing the name of the value. You must provide an 'id' or a 'developerName'.
typeElementId String: GUID (Optional) The unique identifier for the object type of the value in the flow being assigned. Using the typeElementId ensures that your player is guaranteed to be referencing the correct object type. If you use the typeElementDeveloperName property, a change in the type name can break the integration. We ask that you specify the type as we may in future support type casting and we therefore want to know the type you're passing in so we can validate it is correct before mapping to the super type.
typeElementPropertyId String: GUID The unique identifier for the object property of the value in the flow being assigned. Typed objects all have properties. Using the typeElementPropertyId allows you to assign a property in an object value as opposed to the whole value. As with the other identifier references, using the typeElementPropertyId ensures that your player is guaranteed to be assigning the correct value. Using the typeElementPropertyDeveloperName makes it possible to break the integration if the author changes the developerName of a property in the type.
typeElementDeveloperName String (Optional) The developer name for the object type of the value in the flow being assigned. See the typeElementId for more information.
typeElementPropertyDeveloperName String The developer name for the object property of the value in the flow being assigned. See the typeElementPropertyId for more information.
contentValue String The actual content value being assigned to the value in the flow. This property should be used for all "primitive", non-typed values in your flow. For example, if you are referencing a value called "First Name", this would be the value you actually want to assign to it: e.g. "Steve".
contentType String: ContentType The content type of the value you are passing into the flow. The content types are specified by the ContentType enumeration.
id Object: Object Data Object The actual object data being assigne to the value in the flow. If you are assigning a typed object, you should use this property to assign the value. The object data is provided as a list of objects regardless of whether you're assigning an object or a list. However, for objects you should only ever have one object in your array.

API Enumerations Helpful values

InvokeType Enumeration

The InvokeType enum contains the following values:

Value Description
FORWARD Tells the engine that you wish to move forward. This doesn't necessarily mean the engine will move forward if the request fails on page validation OR the author of the flow has put rules against the element that have not evaluated to 'true'.
DONE This invoke type is returned by the engine if the flow has completed.
WAIT Tells the engine and the caller that the engine is waiting or the external plugin is waiting. As part of the plugin framework a "WAIT" request allows you to send status messages to the user without progressing the flow. When calling the engine from a plugin, if the engine responds with a "WAIT", it means the engine is likely waiting on another event or business rule to fire.
STATUS Tells the engine that you wish to get a status update on the flow. This will send back up-to-date page metadata and any new status messages.
SYNC Tells the engine that you wish to synchronize the data held in the player with that held on the service. This will send back up-to-date page metadata and any new status messages.
SUCCESS When making asynchronous requests against the engine, a "SUCCESS" response tells the caller that the request was accepted and the caller therefore does not need to continue to make requests against the engine.
TOKEN_COMPLETED When making asynchronous requests against the engine, a "TOKEN_COMPLETED" response tells the caller that the request is no longer needed or valid and the caller therefore does not need to continue to make requests against the engine. This can happen if the user has send messages out to multiple systems and the rules to continue forward in the execution of the flow were satisfied by another system response.

PageActionBinding Enumeration

The PageActionBinding enum contains the following values:

Value Description
SAVE The engine should apply all values collected to the state.
NO_SAVE The engine should ignore all values collected and not apply anything to the state except the outcome action.

ContainerType Enumeration

The ContainerType enum contains the following values:

Value Description
VERTICAL_FLOW Child page components and containers should be organized vertically.
HORIZONTAL_FLOW Child page components and containers should be organized horizontally.
INLINE_FLOW Child page components and containers should be organized inline (meaning they should appear side-by-side and wrap if there's not enough space horizontally.
GROUP Child page components and containers should be organized as a group. The default runtime interprets a "GROUP" as tabs.

ComponentType Enumeration

The ComponentType enum contains the following values:

Value Description
CONTENT The component collects HTML5 formatted content.
INPUT The component is a field input collecting anything from dates to passwords to text.
TEXTAREA The component supports unformatted, multi-line content.
SELECT The component allows the user to select from a list of options provided by the supplied object data or object data request.
CHECKBOX The component is a simple Yes/No checkbox.
TABLE The component allows the user to select, edit and delete entries in a list of options provided by the supplied object data or object data request.
PRESENTATION The component presents HTML5 formatted content to the user.
TAG The component is unknown and the player should refer to the tag information to determine how to display the provided value or object data.
IMAGE The component is an image reference.

ContentType Enumeration

The ContentType enum contains the following values:

Value Typed Description
ContentString No A Java java.lang.String or C# System.String value.
ContentNumber No A Java java.lang.Double or C# System.Double value.
ContentBoolean No A Java java.lang.Boolean or C# System.Boolean value.
ContentPassword No A Java java.lang.String or C# System.String value. If you indicate a value is a password, we use additional value encryption on storage.
ContentDateTime No A Java java.lang.DataTime or C# System.DateTime. Using the REST API, the value should be provided in JavaScript 1.8.5/ECMAScript 5th Edition format.
ContentContent No A Java java.lang.String or C# System.String value. If you indicate a value is "Content", we typically expect that the content holds HTML formatted String content. The engine will also parse the contents of this String with any value references.
ContentObject Yes The value is Object Data based on the ManyWho format. This allows you to pass complex data into ManyWho for processing, including child objects and lists. Any Object Data being passed into the system also needs to have a Type as defined by your Type Elements.
ContentList Yes The value is a list of Object Data based on the ManyWho format. This allows you to pass lists of complex data into ManyWho for processing (of the same Type), including child objects and lists. Any Object Data being passed into the system also needs to have a Type as defined by your Type Elements.
